How I will start

I will say:

Today I am teaching five Bootstrap components: Spinners, Placeholders, Tooltips, Popovers, and Toasts. These are used when the website is loading something, giving small hints, showing extra information, or notifying the user after an action.

Demo file relation: The student demo has five sections in the same order. I should explain each section, then show the code pattern.

1. Bootstrap Spinners

Related demo section: Student demo → “1. Bootstrap Spinners”
I will say:

Spinners are loading indicators. They tell users that something is still processing. Bootstrap gives two main types: border spinner and growing spinner.

Bootstrap variations covered in demo

  • .spinner-border → normal circular border loader
  • .spinner-grow → growing/fading loader
  • Color utilities → .text-primary, .text-success, .text-danger, etc.
  • Small size → .spinner-border-sm, .spinner-grow-sm
  • Button loader → spinner inside button
  • Accessibility → role="status" and .visually-hidden

Code I should explain

<div class="spinner-border text-primary" role="status">
  <span class="visually-hidden">Loading...</span>
</div>

Real-world examples not shown in demo

  • Payment processing: show a spinner on “Pay Now” after user clicks, so they do not click multiple times.
  • Search loading: show spinner while products, jobs, or users are being fetched from the server.

Question I can ask students

Where should we show a spinner in a real website: before data loads or after data is already visible?

2. Bootstrap Placeholders / Skeleton Loading

Related demo section: Student demo → “2. Bootstrap Placeholders / Skeleton Loading”
I will say:

Placeholders are skeleton screens. They are better than blank screens because the user gets an idea of the page layout while data is loading.

Bootstrap variations covered in demo

  • Card skeleton using .placeholder
  • Width using grid columns → .col-12, .col-8, .col-6, .col-4
  • Colors using background utilities → .bg-primary, .bg-success, etc.
  • Sizes → .placeholder-lg, default, .placeholder-sm, .placeholder-xs
  • Animations → .placeholder-glow and .placeholder-wave
  • Disabled placeholder button using .disabled

Code I should explain

<p class="placeholder-glow">
  <span class="placeholder col-7"></span>
  <span class="placeholder col-4"></span>
  <span class="placeholder col-6"></span>
</p>

Real-world examples not shown in demo

  • Instagram/LinkedIn feed: show skeleton cards before posts appear.
  • Dashboard analytics: show placeholder blocks while charts and numbers load from API.

Question I can ask students

Why is a placeholder sometimes better than a spinner for large content areas?

3. Bootstrap Tooltips

Related demo section: Student demo → “3. Bootstrap Tooltips”
I will say:

Tooltips are small help messages. They are best for short text, like explaining an icon or a form field. They need Bootstrap JavaScript initialization.

Bootstrap variations covered in demo

  • Placements → top, right, bottom, left using data-bs-placement
  • Tooltip text using data-bs-title
  • HTML tooltip using data-bs-html="true"
  • Custom tooltip using data-bs-custom-class
  • Disabled button tooltip using wrapper with tabindex="0"
  • JavaScript initialization using new bootstrap.Tooltip()

Code I should explain

<button class="btn btn-outline-primary"
  data-bs-toggle="tooltip"
  data-bs-placement="top"
  data-bs-title="Tooltip on top">
  Top
</button>
const tooltipTriggerList = document.querySelectorAll('[data-bs-toggle="tooltip"]');
[...tooltipTriggerList].map(el => new bootstrap.Tooltip(el));

Real-world examples not shown in demo

  • Password info: show “Use 8+ characters with number and symbol” near an info icon.
  • Dashboard icons: show “Revenue”, “Orders”, or “Pending” explanation when hovering over small icons.

Question I can ask students

Should we put long paragraphs inside tooltips? Why or why not?

4. Bootstrap Popovers

Related demo section: Student demo → “4. Bootstrap Popovers”
I will say:

Popovers are like bigger tooltips. They can have a title and body content. Use them when the user needs more explanation, but we do not want to open a full modal.

Bootstrap variations covered in demo

  • Basic popover using data-bs-toggle="popover"
  • Title using data-bs-title
  • Body using data-bs-content
  • Placements → top, right, bottom, left
  • Dismissible popover using data-bs-trigger="focus"
  • Custom popover using data-bs-custom-class
  • Disabled button popover using wrapper with tabindex="0"
  • JavaScript initialization using new bootstrap.Popover()

Code I should explain

<button class="btn btn-primary"
  data-bs-toggle="popover"
  data-bs-title="Popover title"
  data-bs-content="This is the popover body content.">
  Click for popover
</button>
const popoverTriggerList = document.querySelectorAll('[data-bs-toggle="popover"]');
[...popoverTriggerList].map(el => new bootstrap.Popover(el));

Real-world examples not shown in demo

  • Pricing page: explain what “Enterprise plan” includes when user clicks an info button.
  • Form help: explain what documents are accepted for upload without leaving the page.

Question I can ask students

When should we use a popover instead of a tooltip?

5. Bootstrap Toasts

Related demo section: Student demo → “5. Bootstrap Toasts”
I will say:

Toasts are small notifications. They are used after an action, like save, delete, upload, message sent, or error. They do not block the page like modals.

Bootstrap variations covered in demo

  • Live toast triggered by button
  • Basic toast using .toast and .toast-header
  • Toast body using .toast-body
  • Close button using data-bs-dismiss="toast"
  • Positioning using .toast-container, .position-fixed, .bottom-0, .end-0
  • Stacked toasts inside .toast-container
  • Custom content toast using .text-bg-success and .d-flex
  • JavaScript trigger using bootstrap.Toast.getOrCreateInstance()

Code I should explain

<button type="button" class="btn btn-primary" id="liveToastBtn">Show live toast</button>

<div class="toast-container position-fixed bottom-0 end-0 p-3">
  <div id="liveToast" class="toast" role="alert" aria-live="assertive" aria-atomic="true">
    <div class="toast-header">
      <strong class="me-auto">Bootstrap</strong>
      <button type="button" class="btn-close" data-bs-dismiss="toast"></button>
    </div>
    <div class="toast-body">This is a live toast notification.</div>
  </div>
</div>
const toastTrigger = document.getElementById('liveToastBtn');
const liveToast = document.getElementById('liveToast');
const toastBootstrap = bootstrap.Toast.getOrCreateInstance(liveToast);
toastTrigger.addEventListener('click', () => toastBootstrap.show());

Real-world examples not shown in demo

  • E-commerce: show “Item added to cart” after the user clicks Add to Cart.
  • Admin dashboard: show “User deleted successfully” or “Settings saved” after an action.

Question I can ask students

Why is a toast better than an alert for temporary success messages?

Final recap I can say

Spinners show loading. Placeholders show the shape of content while loading. Tooltips explain small things. Popovers explain bigger things. Toasts notify the user after an action. These components improve user experience because users always know what is happening.
